## History 路由和 Hash 路由

前端路由的两种核心实现方式是 History 模式和 Hash 模式。它们都是为了解决单页面应用（SPA）在前端路由中的历史记录管理和导航问题。

**History 模式**： History 模式通过 HTML5 提供的 History API 实现。它利用浏览器的 history.pushState() 和 history.replaceState() 方法来操作浏览器历史记录，从而实现前端路由的切换和导航。

- 原理： 当路由切换时，History 模式会调用 pushState() 或 replaceState() 方法将新的路由信息添加或替换到浏览器历史记录中。这样，用户可以通过浏览器的前进和后退按钮进行导航，同时不会发送真正的 HTTP 请求。
- URL 格式： History 模式的 URL 格式是基于真实的 URL 路径，不包含 # 符号，例如：http://example.com/user/profile。
- 服务器配置： 为了支持 History 模式，服务器需要配置一个通配符的路由，将所有的请求都指向 SPA 的入口文件，然后由前端路由根据 URL 来渲染相应的页面。

**Hash 模式**： Hash 模式通过修改 URL 中的哈希（即 #）来实现路由的切换和导航。在改变哈希的同时，页面并不会重新加载，因此适合在老版本浏览器中使用。

- 原理： 当路由切换时，Hash 模式会修改 URL 中的哈希部分，并触发浏览器的 hashchange 事件。通过监听该事件，前端路由可以捕获到路由变化，并进行相应的页面渲染。
- URL 格式： Hash 模式的 URL 格式是在基本路径后加上 # 符号和路由地址，例如：http://example.com/#/user/profile。
- 服务器配置： 由于 Hash 模式的 URL 不会发送到服务器，因此不需要任何特殊的服务器配置。

总结一下， History 模式和 Hash 模式都是前端常见的路由实现方式。History 模式使用真实的 URL 路径，兼容性较好，但需要服务器配置支持；Hash 模式使用哈希来模拟路由，兼容性更好，无需服务器配置。选择哪种实现方式要根据项目需求和浏览器兼容性进行选择。



## Vue-router

Vue Router 是 [Vue.js](https://vuejs.org/) 的官方路由。它与 Vue.js 核心深度集成，让用 Vue.js 构建单页应用变得轻而易举。功能包括：

- 嵌套路由映射
- 动态路由选择
- 模块化、基于组件的路由配置
- 路由参数、查询、通配符
- 展示由 Vue.js 的过渡系统提供的过渡效果
- 细致的导航控制
- 自动激活 CSS 类的链接
- HTML5 history 模式或 hash 模式
- 可定制的滚动行为
- URL 的正确编码

Vue-router 是 Vue.js 官方提供的路由管理器，它解决了以下几个问题：

1. 单页面应用（SPA）的前端路由：Vue-router 实现了前端路由，允许在单页面应用中通过 URL 的变化来实现页面的切换和导航，而不需要每次切换页面都重新加载整个页面。
2. 嵌套路由和视图：Vue-router 支持嵌套路由和多视图的功能，可以构建更复杂的页面结构和布局。通过配置路由的嵌套关系，可以在一个父级路由下包含多个子级路由，并在相应的视图容器中渲染子级组件。
3. 路由参数和查询参数：Vue-router 提供了动态路由参数和查询参数的支持。可以通过配置路由规则来捕获 URL 中的动态参数，并将其作为组件的 props 传递。同时，还可以通过路由的 query 字段传递查询字符串参数。
4. 导航守卫：Vue-router 提供了全局的导航守卫和路由独享的导航守卫，用于在路由切换前后执行一些逻辑。可以通过导航守卫来进行权限验证、重定向等操作，保障页面的安全性和正确性。

Vue-router 的核心原理主要包括以下几点：

1. 核心 API：Vue-router 通过提供一系列核心的 API，例如：Router、Route、RouterView 等，来实现路由的配置和管理。其中，Router 负责整个路由系统的初始化和配置，Route 表示单个路由的对象，RouterView 是用于渲染当前路由对应的组件。
2. 响应式数据：Vue-router 使用 Vue.js 的响应式机制来监听 URL 的变化，并根据变化更新路由状态。通过监听浏览器的 popstate 事件或 hashchange 事件，Vue-router 可以捕获 URL 的变化并及时更新路由状态。
3. 动态路由匹配：Vue-router 会根据路由的配置规则来进行动态路由的匹配。当 URL 发生变化时，Vue-router 会根据配置的路由规则，匹配相应的路由记录，并将匹配结果作为路由状态保存。
4. 组件渲染：Vue-router 利用 Vue.js 的动态组件特性，在 RouterView 组件中根据当前路由状态来动态渲染对应的组件。根据路由记录的组件名称或路径，Vue-router 会创建对应的组件实例，并将其插入到合适的位置。

总结： Vue-router 解决了单页面应用中的前端路由问题，提供了嵌套路由、路由参数、查询参数等功能，并支持导航守卫。其核心原理包括核心 API、响应式数据、动态路由匹配和组件渲染。通过这些特性和原理，Vue-router 实现了灵活、高效的前端路由管理。
